Preparing Your Autohotkey Include Files for Obfuscation

This obfuscator does not automatically find and parse your Autohotkey include files and make them part of the obfuscation. First you must move your '#include <file>' statements in your source code to the beginning or end of your source code files and use the 'ignore before this' and 'ignore after this' obfuscator command comments. Then to signify to the obfuscator what files are to be included, you prepare an includes map file.

Move Your '#include <file>' Statements to the Beginning or End of Your Source Code Files

Move your '#include <file>' statements to the beginning of your source code file and put this obfuscator command comment after them:
;$OBFUSCATOR: $IGNORE_BEFORE_THIS:

Or you can move them to the end of your source code file and put this command comment before them:
;$OBFUSCATOR: $IGNORE_AFTER_THIS:

The obfuscator will not try to process anything before or after those tags in the current source code file. No obfuscation of those areas will be done and in fact none of the code there will be transferred to the final obfuscated code. This must be done with all your '#include <file>' statements.

If your include statements only bring in whole functions or label sections then moving them around should not pose a problem. If you are, however, using includes to drop code fragments inline into other stuff and cannot move them around, then that feature is not supported by this obfuscator.

Prepare an Includes Map File

You tell the obfuscator what files to obfuscate together by preparing an includes map file indicating your main script and all your includes. Just prepare a pure text file with the file path for each file on a separate line. You will be asked for this file when you run the obfuscator. The file that has your autoexecute section on it should be listed first. The order that the remaining files are listed can make a difference because if there is an obfuscator command comment to change obfuscation defaults for functions, then that change will remain in effect across any new source code files until a new command comment changing function defaults is found.
[DigiDon] The Include Map File can now contain relative files, directories (to change relative paths), and comments on separated lines.

If you have a single MainScript.ahk script file with no includes

You still should create an '[SCRIPTNAME]_IncludeMAP.txt' file with in the same folder as your MainScript containing only:
MainScript.ahk

If you have a script including other .ahk files

For example if you have at the end of your MainScript.ahk :
;LIBRARIES
#include Lib\Init.ahk
#include Lib\Init_Settings.ahk
#include Lib\Init_Variables_PreInit.ahk
;INCLUDES
#include Includes\XML_Builder.ahk

You should put before
;$OBFUSCATOR: $IGNORE_AFTER_THIS:

And create an '[SCRIPTNAME]_IncludeMAP.txt' file with :
YOU MUST FIRST LIST OUR MAIN SCRIPT, path can be relative to this _IncludeMAP. So if you have created this file in the same folder as your MainScript, this file can contains:
MainScript.ahk
;LIBRARIES
Lib\Init.ahk
Lib\Init_Functions.ahk
;INCLUDES
Includes\XML_Builder.ahk
